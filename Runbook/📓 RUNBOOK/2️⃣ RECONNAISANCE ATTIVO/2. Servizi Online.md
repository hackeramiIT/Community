### Identificare le versioni
Per identificare la versione di un servizio si possono utilizzare diversi strumenti e/o metodi:
- *Codice Sorgente:* ispezionando il codice sorgente con CTRL+U, spesso appaiono le versioni associate al servizio/plugin utilizzato dall'applicazione.
- *Wappalyzer:* un plugin molto utile scaricabile gratuitamente sul proprio browser dal seguente [link](https://chromewebstore.google.com/detail/wappalyzer-technology-pro/gppongmhjkpfnbhagpmjfkannfbllamg?hl=en&pli=1).
- *Richieste HTTP/S:* intercettando le richieste HTTP/S si potrebbero trovare le versioni dei servizi associati. Per l'intercettazione possiamo utilizzare la seguente [guida](https://portswigger.net/burp/documentation/desktop/getting-started/intercepting-http-traffic).
- *Misconfiguration:* pagine, input o directory mal gestite potrebbero rivelarci informazioni importanti come versione dei servizi o [[ðŸ”¡ Glossario#Stack Trace| stack traceÂ°]].
- *Scanner appositi:* in alcuni casi tools classici non sono sufficienti per una scansione del servizio, dunque dobbiamo utilizzare quelli appositi (es: `wpscan` per [[Wordpress]]).
- L'uso di *tool generici* come:

```
nikto -h IP
```

